## Build & Run Commands

This is a native iOS app built with Swift/Xcode.

```bash
# Build (Debug)
xcodebuild -scheme BabyCryAnalyzer -configuration Debug build

# Run on simulator
xcodebuild -scheme BabyCryAnalyzer -configuration Debug -destination 'platform=iOS Simulator,name=iPhone 15' run

# Run unit tests
xcodebuild -scheme BabyCryAnalyzerTests test

# Run UI tests
xcodebuild -scheme BabyCryAnalyzerUITests test
```

Open `ios/BabyCryAnalyzer.xcodeproj` in Xcode 16+ for interactive development.

## Architecture Overview

**Type:** Native iOS app (Swift 6.0, SwiftUI, iOS 16+ deployment target)
**Purpose:** Record or upload audio of baby cries, classify the cry type via AI, and maintain a local history.

### Layer Structure

```
ios/BabyCryAnalyzer/
├── Services/       — External dependencies & business logic
├── ViewModels/     — @Observable state bridging services to views
├── Views/          — SwiftUI views (pure UI)
├── Models/         — Data types (CryAnalysis, CryReason enum)
└── AppConfig.swift — Runtime configuration constants
```

### State Management

Uses Swift 6.0 `@Observable` macro throughout (not `ObservableObject`/`@Published`):
- `AuthService` — Supabase session state, injected via `.environment()` at app root
- `ListenViewModel` — Primary recording/analysis state for the Listen tab
- `AudioRecordingService` — AVAudioRecorder state and metering
- `CryHistoryStore` — Local JSON persistence per user

### Analysis Pipeline

1. **Record/Upload** → `AudioRecordingService` or `AudioFileAnalyzer` extracts duration & dB metrics
2. **Transcribe** (optional) → `SpeechTranscriptionService` POST to `/stt/transcribe/`
3. **AI Analysis** → `CryAnalysisService` POST to `/agent/chat` → returns `{reason, confidence, tip}` JSON
4. **Fallback** → Local heuristic analysis if server unavailable (2 retries with backoff)
5. **Persist** → `CryHistoryStore` saves to `~/Library/Application Support/cry_analyses_{userId}.json`

### External Services

- **Supabase** (`cmtlwxpqgrslnknlmraf.supabase.co`) — Authentication only
- **Rork Toolkit** (`toolkit.rork.com`) — AI agent chat (`/agent/chat`) and speech-to-text (`/stt/transcribe/`)
  - Requests include `x-project-id` and `x-team-id` headers from `Config.swift`

### Configuration

`Config.swift` is generated by Rork CI/CD and `.gitignore`'d. It provides:
- `EXPO_PUBLIC_SUPABASE_URL` / `EXPO_PUBLIC_SUPABASE_ANON_KEY`
- `EXPO_PUBLIC_TOOLKIT_URL` — base URL for AI and STT endpoints
- `EXPO_PUBLIC_PROJECT_ID` / `EXPO_PUBLIC_TEAM_ID`

Fallback defaults are embedded in `AppConfig.swift` for local development without `Config.swift`.

### Concurrency Model

All services use `async/await`. UI-bound state is isolated with `@MainActor`. Audio metering runs on a 50ms `Timer` on the main thread.
